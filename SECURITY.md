# Security Policy

## Supported Versions

Security fixes are applied to the latest `main` branch.

## Reporting a Vulnerability

Do not open public issues for vulnerabilities.

Instead:

1. Contact maintainers privately with a reproduction and impact summary.
2. Include affected files/routes/commands and mitigation ideas if available.
3. Allow time for triage and patching before disclosure.

## Scope

Security-sensitive areas in this repository include:

- Bot token and API token handling.
- Claim/spend submission paths.
- Input validation for Discord links and command payloads.
- Adapter network calls and error handling.

## Response Goals

- Initial acknowledgment within 72 hours.
- Triage and severity classification as quickly as practical.
- Patch and release plan shared privately with reporter.
